# VTT Analysis Website — 完整方案

## 目标

搭建一个**公开可访问的分析网站**，用户输入 YouTube 链接后：
1. 自动下载视频字幕（VTT）
2. 转换为干净的 Markdown 文本
3. 生成可分享的分析报告页

定位：**字幕分析 + 分享工具网站**，演示 vYtDL 的核心能力。

---

## 架构总览

```
用户浏览器（Next.js 静态页面）
     ↓  HTTP API / WebSocket
Express Web Server（已有 web-server/）
     ↓  spawn()
yt-dlp（下载字幕）  +  Go CLI analyze（VTT → Markdown）
     ↓  结果写入
SQLite（持久化）  +  静态文件（Markdown 输出）
```

**核心思路**：复用已有的三层架构，新增两个模块：

| 层 | 已有 | 新增 |
|----|------|------|
| 前端 (Next.js) | `home`, `library`, `player`, `settings` | `analyze` 页面 + 分享页 |
| 后端 (Express) | download API, queue, settings | VTT analyze API |
| CLI (Go) | `analyze --mode text/markdown` | **无需改动** ✅ |

---

## 功能模块

### Module A: 输入 → 分析流程

```
用户输入 YouTube URL
    ↓
后台 yt-dlp --skip-download --write-subs --write-auto-subs
    ↓
拿到 .vtt 文件
    ↓
Go CLI: vYtDL analyze --mode markdown xxx.vtt
    ↓
生成 Markdown 文本 + 存入 SQLite
    ↓
重定向到分析报告页
```

### Module B: 分析报告页（可分享）

每个分析结果有唯一 ID，可通过 URL 分享：

```
https://vtt-demo.example.com/analyze/<report-id>
```

报告页展示：
- 视频标题 + 缩略图
- 语言标签（zh/en/...）
- **完整 Markdown 字幕**（时间戳 + 文本）
- 时长统计
- 一键复制 / 下载 Markdown

### Module C: 历史列表

```
https://vtt-demo.example.com/analyze  （列表页）
```

- 所有已分析的视频列表
- 搜索 / 按语言过滤
- 点击进入报告页

## 数据模型

新增 `vtt_reports` 表：

```sql
CREATE TABLE vtt_reports (
  id          TEXT PRIMARY KEY,      -- UUID
  youtube_url TEXT NOT NULL,
  video_id    TEXT,                  -- YouTube video ID
  title       TEXT,
  language    TEXT,                  -- zh / en / ...
  content     TEXT NOT NULL,         -- Markdown 全文
  cue_count   INTEGER,
  duration_sec REAL,                -- 视频总时长（秒）
  created_at  TEXT DEFAULT (datetime('now')),
  status      TEXT DEFAULT 'done'   -- pending / processing / done / failed
);
```

## API 设计

### `POST /api/analyze-vtt`
请求视频分析：
```json
{ "url": "https://youtube.com/watch?v=xxx" }
```
返回：
```json
{ "success": true, "data": { "reportId": "uuid-xxx" } }
```
后台异步：下载 VTT → 转 Markdown → 入库 → WebSocket 通知前端。

### `GET /api/vtt-report/:id`
获取分析结果：
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "title": "AI的正确打开方式：不学概念，学动作",
    "language": "zh",
    "content": "# Transcript\n\n## 00:00:25\n...",
    "cue_count": 348,
    "duration_sec": 2340,
    "created_at": "..."
  }
}
```

### `GET /api/vtt-reports`
获取所有报告列表（分页）：
```
GET /api/vtt-reports?page=1&limit=20&lang=zh
```

### WebSocket Event: `vtt-report-complete`
报告生成完毕后推送：
```json
{ "type": "vtt-report-complete", "reportId": "uuid" }
```

## 实施步骤

### Phase 1: 后端 API（1-2h）

1. **新增 `vtt_reports` 表到 `database.ts`**
   - `createTable()` 迁移
   - `createReport()` / `getReport()` / `listReports()`

2. **新增 `vtt-analysis.ts` 服务**
   - `analyzeVideo(url)` — 调用 yt-dlp 下载 VTT，再调用 Go CLI 转 Markdown
   - 处理异步流程 + 状态更新

3. **注册 API 路由到 `index.ts`**
   - `POST /api/analyze-vtt`
   - `GET /api/vtt-report/:id`
   - `GET /api/vtt-reports`

### Phase 2: 前端页面（1-2h）

1. **`/analyze/page.tsx`** — 输入页
2. **`/analyze/[id]/page.tsx`** — 报告展示页
3. **组件**：`AnalyzeForm`, `ReportViewer`, `RecentList`

### Phase 3: Docker 集成（0.5h）

1. 将 Go CLI 二进制添加到 Docker 镜像
2. 或通过环境变量指定 CLI 路径

### Phase 4: 部署 & 分享（0.5h）

1. 生成 `og:image` 社交预览卡
2. SEO meta tags
3. 部署到服务器 / VPS

## 技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| VTT 解析 | **Go CLI** `analyze --mode markdown` | 已有完整实现，零开发成本 |
| 前端框架 | **Next.js**（复用现有） | 已有静态导出 + SSR 能力 |
| 后端 | **Express**（复用现有） | 已有 download/queue/SQLite 基建 |
| 数据库 | **SQLite**（复用现有） | 已有 better-sqlite3 |
| 部署 | **Docker**（复用现有） | 已有 Dockerfile + compose |
