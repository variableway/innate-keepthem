# Desktoplize 项目分析报告

## 项目概述

将现有的 vYtDL (Go CLI YouTube Downloader) 转换为基于 Tauri + shadcn-ui + TypeScript 的桌面应用程序。

---

## 一、功能需求分析

### 核心功能 (P0 - 必须实现)

| 序号 | 功能 | 可行性 | 技术难度 | 说明 |
|------|------|--------|----------|------|
| 1 | URL 输入下载 | ✅ 高 | 低 | 前端表单 + Tauri Command 调用现有 Go 下载器 |
| 2 | 文件下载 | ✅ 高 | 中 | 复用现有 Go downloader 模块 |
| 3 | 下载进度显示 | ✅ 高 | 中 | Tauri Event 通道 + 前端进度条组件 |
| 4 | 下载状态显示 | ✅ 高 | 低 | 状态管理 + UI 反馈 |
| 5 | 视频和字幕下载 | ✅ 高 | 低 | 现有功能，通过 options 控制 |
| 6 | 下载位置通知 | ✅ 高 | 低 | Tauri Notification API |
| 7 | 下载文件浏览 | ✅ 高 | 中 | 本地数据库 + 列表视图 |

### 扩展功能 (P1 - 增值功能)

| 序号 | 功能 | 可行性 | 技术难度 | 说明 |
|------|------|--------|----------|------|

| 层级 | 技术 | 版本 | 理由 |

## 四、存在的问题与解决方案

### 4.1 现有代码库问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| CLI 耦合度高 | downloader 直接输出到 stdout/TUI | 抽象 `ProgressSink` 接口，支持多种输出 |
| 状态存储简单 | 仅 JSON 文件，无查询能力 | 增加 SQLite 存储层，保留 JSON 备份 |
| 无 API 接口 | 仅命令行参数 | 包装为可导入的 Go 库或 HTTP 服务 |
| 硬编码配置 | config.json 路径固定 | 增加配置管理接口 |

### 4.2 技术挑战

#### 挑战 1: Go-Rust 通信

**现状**: 现有代码是纯 CLI，通过 stdout/stderr 交互。

**方案对比**:

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| A. Subprocess | 简单，零改造 | 进程管理复杂 | ✅ 推荐 |
| B. CGO 绑定 | 性能高 | 复杂，跨平台难 | ❌ |
| C. gRPC/HTTP | 标准化 | 需要额外端口 | ⚠️ 备选 |

**决策**: 采用 **方案 A (Subprocess)**，理由：
- 改动最小，快速上线
- 利用 yt-dlp 本身的 stdout 进度输出
- 易于调试和替换

#### 挑战 2: 实时进度传输

```
Frontend ←──tauri:event──┐
                         ├─ Tauri Backend
Go Downloader ──stdout───┘
```

实现步骤:
1. Tauri 启动 Go 下载器作为子进程
2. 解析 stdout 中的进度 JSON
3. 通过 `tauri::Emitter` 发送事件到前端
4. 前端监听事件更新 UI

#### 挑战 3: 视频播放方案

| 方案 | 支持格式 | 字幕支持 | 打包大小 | 推荐 |
|------|----------|----------|----------|------|
| HTML5 Video | 有限 | 一般 | 小 | ⚠️ 备选 |
| Plyr | 广泛 | 好 | 中 | ✅ 推荐 |
| Video.js | 广泛 | 好 | 大 | ⚠️ 备选 |
| 系统播放器 | 最全 | 依赖系统 | 零 | ❌ 体验差 |

**决策**: 使用 **Plyr**，理由：
- 现代化 UI，与 shadcn 风格匹配
- 良好的字幕支持
- 适中的打包大小

## 六、结论与建议

### 6.1 可行性结论

✅ **项目可行**，建议推进

- 核心功能技术成熟，风险可控
- 现有 Go 代码可复用，节省开发时间
- Tauri 生态完善，社区支持好

### 6.2 关键成功因素

1. **渐进式迁移**: 先保证 CLI 功能完整移植，再添加新功能
2. **接口抽象**: 尽早定义好 Frontend-Backend-Core 的通信协议
3. **用户体验**: 桌面应用的优势在于交互，要充分利用
