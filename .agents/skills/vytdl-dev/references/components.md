# Component Development Reference

## CLI (vYtDL/)

### Adding a New Flag

1. Add flag variable in `cmd/download.go` `init()`
2. Add field to `downloader.Options` struct
3. Pass through to yt-dlp args in `internal/downloader/downloader.go`

### Key Files
- `cmd/root.go` - Root command
- `cmd/download.go` - Flag parsing, TUI coordination
- `internal/downloader/downloader.go` - Core yt-dlp wrapper
- `internal/tui/` - Bubble Tea terminal UI
- `internal/record/` - Download tracking files

## Desktop App (vYtDL-desktop/apps/desktop/)

### Frontend Structure

```
src/
  app/              # Next.js pages (home, settings, library, player)
  components/       # React components
    download-form.tsx    # Single/Batch/Smart download input
    download-list.tsx    # Download list with actions
    layout/              # app-shell, app-sidebar
  i18n/             # Translation system
    locales/        # en.json, zh.json, ja.json
  lib/
    api-client.ts   # Tauri IPC / HTTP abstraction
  store/
    downloadStore.ts
    settingsStore.ts
```

### Adding a Language

1. Copy `en.json` to new locale file
2. Translate all values
3. Import in `src/i18n/index.tsx`
4. Add option in `src/app/settings/page.tsx`

### Download Form Modes

| Mode | Input | Behavior |
|------|-------|----------|
| single | Single `<Input>` | Fetches video info on URL input |
| batch | `<textarea>` + file import | Parses multiple URLs |
| smart | `<textarea>` + file import | Auto-detects playlist URLs |

### Rust Backend (src-tauri/src/)

| File | Purpose |
|------|---------|
| `commands.rs` | All IPC command handlers |
| `downloader.rs` | yt-dlp subprocess wrapper |
| `database.rs` | SQLite database layer |
| `queue.rs` | Async download queue manager |
| `lib.rs` | App setup, auto-resume on startup |
| `audio_extractor.rs` | FFmpeg audio extraction |

## Web Server (vYtDL-desktop/web-server/)

### Adding an Endpoint

1. Add route in `src/index.ts`
2. Implement logic using `database.ts` and `downloader.ts`
3. Ensure desktop frontend uses same command name via `apiInvoke()`

### Key Files
- `src/index.ts` - Express routes
- `src/downloader.ts` - yt-dlp and FFmpeg wrappers
- `src/queue.ts` - In-memory queue manager
- `src/database.ts` - SQLite wrapper

## Chrome Extension (url-extractor/)

### Key Files
- `manifest.json` - Manifest V3 config
- `popup.html/js/css` - Extension popup UI
- `content.js` - Content script for URL extraction
- `icons/` - Extension icons (generated by `generate_icons.py`)

### Regenerating Icons
```bash
cd url-extractor/icons
python3 generate_icons.py
```
